## 2. 프로젝트 아키텍처 개요

### 2.1 시스템 구조

본 프로젝트는 **모놀리식(Monolithic) 아키텍처**로 구성됩니다.

### 2.2 레이어 구조

프로젝트는 계층형 아키텍처(Layered Architecture)를 따르며, 각 레이어는 명확한 책임을 가집니다.

```
┌─────────────────────────────────────┐
│  Presentation Layer (Controller)    │  ← REST API, DTO
├─────────────────────────────────────┤
│  Application Layer (Service)        │  ← 비즈니스 로직
├─────────────────────────────────────┤
│  Domain Layer (Entity, Repository)  │  ← 도메인 모델
├─────────────────────────────────────┤
│  Infrastructure Layer               │  ← 외부 API, AWS S3, DB
└─────────────────────────────────────┘
```

#### Presentation Layer (Controller)

- REST API 엔드포인트 제공
- 요청/응답 DTO 처리
- HTTP 상태 코드 및 예외 처리
- 입력 유효성 검증

#### Application Layer (Service)

- 비즈니스 로직 구현
- 트랜잭션 관리
- 여러 도메인 객체 조합 및 조율

#### Domain Layer (Entity, Repository)

- 핵심 도메인 모델 정의
- 엔티티 및 값 객체
- Repository 인터페이스 정의
- 도메인 규칙 및 검증 로직

#### Infrastructure Layer

- 외부 시스템 연동 (외부 API, AWS S3 등)
- 데이터베이스 접근 (JPA 구현체)
- 설정 및 공통 유틸리티

## 5. 로깅

### 5.1 로깅 프레임워크

| 항목              | 기술    | 비고                     |
| ----------------- | ------- | ------------------------ |
| Logging Framework | Logback | SLF4J 구현체             |
| Facade            | SLF4J   | `@Slf4j` 어노테이션 사용 |

#### 사용 방법

```java
import lombok.extern.slf4j.Slf4j;

@Slf4j
@Service
public class OrderService {

  public void createOrder(OrderRequest request) {
    log.info("Order creation started: {}", request.getOrderId());

    try {
      // 비즈니스 로직
      log.debug("Processing order: {}", request);
    } catch (Exception e) {
      log.error("Order creation failed: {}", request.getOrderId(), e);
      throw e;
    }

    log.info("Order created successfully: {}", request.getOrderId());
  }
}
```

#### 로그 레벨 가이드

- **ERROR**: 시스템 오류, 즉시 대응 필요
- **WARN**: 잠재적 문제, 모니터링 필요
- **INFO**: 주요 비즈니스 이벤트 (생성, 수정, 삭제 등)
- **DEBUG**: 상세 처리 과정 (개발/디버깅 용도)
- **TRACE**: 매우 상세한 정보 (거의 사용 안 함)
